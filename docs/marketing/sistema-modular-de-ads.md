# Sistema Modular de Ads para Propiedades

Framework de producción para grabar una propiedad una sola vez y salir con suficientes módulos para testear múltiples anuncios en Meta Ads sin rehacer el video completo.

La lógica base es:

`Hook` + `Ángulo` + `Prueba visual de la propiedad` + `CTA`

Si se mantienen constantes los visuales base de la propiedad y el CTA, `3 hooks x 3 ángulos` producen `9` combinaciones reales para testing.

## Cuándo usar este sistema

Usarlo cuando:

* se va a grabar una casa, modelo o proyecto y conviene salir con varias versiones de anuncio
* se quiere testear hooks y ángulos sin cambiar la propiedad ni el destino del lead
* se necesita un flujo repetible para listings, new construction o comunidades enteras

No usarlo como excusa para meter demasiadas variables. Si también cambias CTA, duración, oferta, formato y edición, luego no sabrás qué ganó.

## Estructura del sistema

### Variables

* `Hooks`: primeras `1-2` frases que compiten por detener el scroll
* `Ángulos`: razón principal por la que la persona debería interesarse

### Constantes

* `Visuales base`: frente, entrada, cocina, sala, cuarto, baño, patio, detalles
* `Prueba`: números, doble acceso, acabados, incentivos, ubicación, extras
* `CTA`: mensaje final y destino del lead

## Hooks recomendados

Los hooks deben competir entre sí. No deben ser versiones casi idénticas.

Ejemplos:

1. `Entrada baja` `Con $13,000 podrías entrar a esta propiedad.`
2. `Ingreso` `Esta casa tiene dos puertas y eso cambia todo.`
3. `Salir de renta` `Si estás rentando, mira esto antes de renovar contrato.`

## Ángulos recomendados

Cada ángulo debe desarrollar una motivación distinta.

Ejemplos:

1. `Entrada baja` Explicar por qué la barrera de entrada es menor de lo que la persona imagina.
2. `Vive + genera ingreso` Explicar por qué la distribución o el anexo abren una segunda utilidad.
3. `Dejar de rentar` Comparar mentalmente el paso de renta a compra sin inventar pagos o beneficios no confirmados.

## Orden de testing recomendado

### Opción 1: lectura más limpia

1. Testear hooks con el mismo ángulo.
2. Elegir hook ganador.
3. Testear ángulos usando ese hook.

### Opción 2: lanzamiento completo

Lanzar la matriz completa de `9` variantes cuando haya suficiente presupuesto y el equipo pueda leer resultados por combinación.

## Shooting Sheet

Usar esta hoja en campo para no improvisar.

### Bloque A. Hooks

Grabar `5-7` hooks por propiedad aunque después solo se publiquen `3`.

Reglas:

* durar `3-6` segundos
* decir el beneficio principal en la primera frase
* mirar a cámara
* dejar una micro pausa al inicio y al final para edición
* grabar una versión neutra y una versión más agresiva

Plantilla:

```
Hook 1:
Hook 2:
Hook 3:
Hook 4:
Hook 5:
```

### Bloque B. Ángulos

Grabar `3-5` segmentos de venta por propiedad.

Reglas:

* durar `5-10` segundos
* desarrollar una sola idea
* no repetir el hook palabra por palabra
* conectar el mensaje con una parte concreta de la propiedad

Plantilla:

```
Ángulo 1:
Problema o deseo:
Prueba visual que lo respalda:

Ángulo 2:
Problema o deseo:
Prueba visual que lo respalda:

Ángulo 3:
Problema o deseo:
Prueba visual que lo respalda:
```

### Bloque C. Prueba y soporte

Grabar recursos que sostengan los claims.

Checklist:

* frente de la casa
* número o rango de precio si se va a usar
* doble entrada o acceso independiente
* cocina o kitchenette del anexo
* laundry
* sala y comedor
* cuarto principal
* baño principal o secundario
* patio, vista o amenidad
* detalles de acabados
* branding de agente o proyecto

### Bloque D. CTA

Grabar `2-3` cierres con la misma intención.

Ejemplos:

* `Escríbeme "INFO" y te mando los números.`
* `Mándame mensaje y te digo si calificas.`
* `Te explico paso a paso por WhatsApp.`

### Bloque E. B-roll obligatorio

Grabar suficiente variedad para que las `9` piezas no se sientan iguales.

Lista mínima:

* `3` tomas exteriores abiertas
* `3` caminatas entrando
* `2` tomas por espacio principal
* `2` close-ups de detalles
* `1` toma lenta por cada beneficio fuerte
* `1` toma estable para portada o thumbnail

### Bloque F. Captura de audio y texto

* grabar en vertical `9:16`
* usar micrófono o ambiente limpio
* repetir números importantes dos veces
* pedir una toma específica para disclaimers condicionados: `si calificas`, `según tu perfil`, `sujeto a disponibilidad`

## Qué dejar constante

Para que el test sirva, mantener constantes:

* propiedad o proyecto
* duración aproximada
* estilo de edición
* subtítulos
* CTA
* destino del lead

## Qué sí conviene variar

* primer frame
* hook verbal
* texto en pantalla del hook
* ángulo principal
* orden del b-roll para sostener el ángulo

## Guion de grabación en sitio

Usar este orden para no perder tiempo:

1. Exterior y hook principal.
2. Hooks alternos en la misma posición.
3. Entrada y doble acceso.
4. Segmentos de ángulo por espacio relevante.
5. B-roll completo.
6. CTA final.
7. Tomas de seguridad con números y disclaimers.

## Criterios de calidad para aprobar una variante

* el hook se entiende sin contexto previo
* el ángulo no repite el hook de forma vacía
* cada claim tiene soporte visual o fuente confirmada
* el CTA dice qué debe hacer la persona y qué recibirá
* la pieza funciona sin audio gracias a subtítulos y on-screen text

## Errores comunes

* grabar solo un tour largo y luego intentar cortarlo en pauta
* hacer hooks demasiado parecidos
* repetir el mismo b-roll en las `9` versiones
* meter `3` o `4` ideas fuertes dentro del mismo ángulo
* usar números no verificados o incentivos viejos
* dejar el CTA solo al final
